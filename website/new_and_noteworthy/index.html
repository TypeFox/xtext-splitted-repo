Xtend 2.3 BETA New & Noteworthy

Debugging with JSR-045

Debugging through Xtend and Java is now fully supported. Setting breakpoints in Xtend works just as in Java and can even switch to the Java level whenever wished using a simple action.

Also all the cool features known from the Java debugger, like ‘Display’, ‘Inspect’, or ‘Exception Breakpoints’ are available.

Even Better JDT Integration Forget about the differences

Not only the debugger but all of JDT’s navigation functionality now works seamlessly with Xtend. No matter if you click on a stack frame in an exception, a failing unit tests, some node in the call hierarchy or the type hierarchy: if the target element is implemented in Xtend, you’ll end up in the Xtend editor at the original source location.

Generated Code View Feel the Trace

But what if you want to see the generated Java source? You can of course open the java file in an editor, but there’s now an even better tool : The Generated Code view. ￼In that view you see detailed information of which parts in the Java source were derived from which in the Xtend code, simply by marking the corresponding parts.

Improved Hovers Everywhere!

Not only the debugger but all of JDT’s navigation functionality now works seamlessly with Xtend. No matter if you click on a stack frame in an exception, a failing unit tests, some node in the call hierarchy or the type hierarchy: if the target element is implemented in Xtend, you’ll end up in the Xtend editor at the original source location.

Call-Hierarchy Who is your client?

In Xtend the open call hierarchy action is now available, using the same shortcuts as in Java. It shows all calls to the selected method, field or constrcutor. No matter whether they are called from Xtend or Java.

Type-Hierarchy Just like in Java

Also the Type Hierarchy view can be opened and populated from within an Xtend class. Just press ‘F4’ when your cursor sits on a type’s name. It shows the type hierarchy of the selected classes, no matter they are implemented in Java or Xtend.

Inherited members in quick outline More Readability

When pressing CTRL+O in an Xtend editor the quick outline pops up, showing all the members of the currently edited class. Pressing CTRL+O once more will add the members inherited from super classes as well. A search field lets you simply filter by name in order to navigate to any member quickly.

Compiler Improvements More Readability

To make the generated Java code even more readable, the compiler can now inline certain operations. As a result the generated Java code is more idiomatic and therefore more readable. Also some superfluous statements have been removed.

Several Language Enhancements Streamlining

On the language level the next release has a couple of nice improvements and new features as well. Some of them are small but neat changes like, that it’s no longer mandatory to write an empty pair of parenthesis in constructor calls. Others deserve a list entry on their own:

Number Literals Fearless arithemtics with big numbers

Xtend now has comprehensive support for number literals. This includes most of the things you can do in Java, and in addition it has first class support for BigDecimal and BigInteger (including overloaded arithmetic operators). 

new BigDecimal("6.1").subtract(
       new BigDecimal("0.755").multiply(BigDecimal.valueOf(3L)))

In Xtend you can use literals and operators : 

6.1bd - 0.755bd * 3bd

Methods with variable argument lists Varargs

Varargs can now be declared in Xtend methdos as well. The syntax and semantics are just like in Java. Also calling varargs works exactly as in Java. 

def String concat(String... strings) {
	....
}

def void useConcat() {
   concat('can', 'pass', 'any', 'number', 'of', 'strings')
}
