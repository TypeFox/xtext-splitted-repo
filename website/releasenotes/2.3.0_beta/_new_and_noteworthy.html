Whats new in Xtext 2.3 BETA

See what was new in the previous release

Xtext 2.3 BETA offers a lot of new features and improvements. All in all, over 120 bug reports and feature requests have been solved since Xtext 2.2. Please note that some the tickets may be related to Eclipse Xtend. Here is a summary of the most significant enhancements:

Liberation of Xtend

The programming language Xtend moved to its own project under the umbrella of eclipse.tools.

Debug Support

For DSLs from which Java code is being generated, Xtext now supports debugging as defined in JSR-045. All you need to do is define a mapping between your DSL files and the generated Java code. Then, the Eclipse JDT debugger will be fully capable of debugging your DSLs. The power of breakpoints, "step into", "step over", "step out" becomes available in your DSL!

Generated Source View

If you generate source code from your Xtext DSLs, it is now easy to implement a View for your workbench that provides a preview of the how the generated code would look like.

Hover Improvements

The new hover for Xbase based languages comes with a completely new HTML rendering for multiline comments of element declarations. Nearly all documentation annotations known from JDT are usable now. Even generics are fully resolved when using generic typed methods.

Xbase Compiler Improvements

To make the generated Java code even more readable, the compiler can now inline certain operations. As a result the generated Java code is more idiomatic and therefore more readable. Also some superfluous statements have been removed.

Xbase Number Literals

Fearless arithmetics with big numbers: Xtend now has comprehensive support for number literals. This includes most of the things you can do in Java, and in addition it has first class support for BigDecimal and BigInteger (including overloaded arithmetic operators). 

new BigDecimal("6.1").subtract(
       new BigDecimal("0.755").multiply(BigDecimal.valueOf(3L)))

In Xtend you can use literals and operators : 

6.1bd - 0.755bd * 3bd
